Quick refresher on the key concepts from W19D1 before we build on them tonight.

CartPole Environment

Balance a pole on a cart. 4 observations (position, velocity, angle, angular velocity), 2 actions (left/right), +1 reward per step.

Q-Learning Algorithm

Build a table of state-action values. Update Q(s,a) based on rewards and future value estimates.

Hyperparameters

Learning rate, discount factor, epsilon (exploration), number of bins for state discretization.

Running the Starter

Auto-venv setup, live training graph, evaluation with epsilon=0, HTML report generation.

Git Setup

10 minutes

Clone the classroom repository and create your feature branch.

# Clone the repository (your instructor will provide the URL) git clone <your-classroom-repo-url> cd w19d2-qlearning # Switch to your team branch git checkout team-a # Replace with your team name # Create your feature branch git checkout -b feature/learning-rate # Use your improvement name

4

Implementation

30 minutes

Implement your improvement in the starter code.

  1. Open w19d2_starter.py in your code editor
  2. Find your section marked with # ========== MODIFY HERE ==========
  3. Implement your improvement (refer to research-topics.html for code hints)
  4. Test your changes:

Open results/report.html in your browser to see your results!

5

Evaluation & Commit

10 minutes

Record your evaluation score and commit your changes.

  1. Note your evaluation score from the report (Mean Score)
  2. Commit your changes with a descriptive message:
# Stage your changes git add w19d2_starter.py # Commit with a good message git commit -m "Add exponential learning rate decay" # Push to GitHub git push -u origin feature/learning-rate

Good Commit Messages

Start with a verb: "Add softmax exploration strategy", "Increase pole angle bins to 48", "Implement angle-based reward penalty"

6

Create Pull Request

10 minutes

Create a Pull Request on GitHub to merge your changes into the team branch.

  1. Go to your repository on GitHub
  2. Click "Compare & pull request" (or Pull Requests > New)
  3. Set base branch to your team branch (e.g., team-a)
  4. Fill in the PR description with:
## Summary [Describe what you changed and why] ## Changes Made - [List specific code changes] ## Evaluation Results - Baseline: [score before your changes] - After change: [your evaluation score] ## Testing - [x] Ran training with seed=42 - [x] Completed 100-episode evaluation
  1. Request review from a teammate
  2. Submit the PR
7

Code Review

5 minutes

Review one of your teammate's Pull Requests.

  1. Open a teammate's PR on GitHub
  2. Review the code changes (click "Files changed")
  3. Check their evaluation results
  4. Leave constructive feedback:
  • Does the code make sense?
  • Did it improve the score?
  • Any suggestions for improvement?
  1. Approve or Request Changes

Section 5: Help & Troubleshooting

Python not found or version error

Make sure Python 3.8+ is installed. Try python3 --version or python --version.

Git command not found

Install Git from git-scm.com. Restart your terminal after installing.

Permission denied when pushing

Make sure you've accepted the GitHub Classroom assignment and have access to the repo.

Matplotlib window not appearing

The live graph requires a display. If using SSH/remote, add --no-plot flag: python w19d2_starter.py --no-plot

Training scores are very low

Check that your modification doesn't break the algorithm. Test with the original code first to establish a baseline.

Still stuck?

Ask your teammates first, then raise your hand for instructor help!
